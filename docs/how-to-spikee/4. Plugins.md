# 4. Plugins

A Plugin is a Python script that transforms a payload during dataset generation. This is typically used to assess transformation-based jailbreaking techniques, or to modify prompts into a target friendly format.

## 4.1 Applying a Plugin
Spikee has several built-in plugins, which are divided into the following categories: Basic Transformations; Attack Plugins, based on dynamic attack modules; and LLM Driven Plugins, which use an LLM agent.

This is a short list of example plugins, see [Built-In Plugins](../02_builtin.md#built-in-plugins) for a full list:
- **1337** (Basic): Transforms text into "leet speak" by replacing certain letters with numbers or symbols.
- **base64** (Basic): Encodes text using Base64 encoding.
- **splat** (Basic): Obfuscates the text using splat-based techniques (e.g., asterisks '*', special characters, and spacing tricks), to bypass basic filters.
- **best_of_n** (Attack): Implements ["Best-of-N Jailbreaking" John Hughes et al., 2024](https://arxiv.org/html/2412.03556v1#A1) to apply character scrambling, random capitalization, and character noising. 


```bash
# Applying the base64 plugin to the cybersec dataset.
spikee generate --seed-folder datasets/seeds-cybersec-2026-01 --plugins base64

# Applying the best_of_n plugin, with 5 variations per input.
spikee generate --seed-folder datasets/seeds-cybersec-2026-01 --plugins best_of_n --plugin-options "best_of_n:variants=5"

# Applying multiple plugins.
spikee generate --seed-folder datasets/seeds-cybersec-2026-01 --plugins 1337 base64 splat
```

**Plugin Piping**
Spikee includes support for plugin piping, allowing you to apply multiple sequential plugins, which can aid more complex jailbreaking techniques.

```bash
# Applying the splat plugin, then the base64 plugin.
spikee generate --seed-folder datasets/seeds-cybersec-2026-01 --plugins splat|base64
```

## 4.2 Creating a Plugin

To create a plugin, extend the `Plugin` or `BasicPlugin` class:
- `Plugin`: Standard plugin implementation, exposing the `transform` function.
- `BasicPlugin`: Simplified extension of the plugin implementation, that exposes the `plugin_transform` function to transform the input text and abstract the exclude patterns. (See [implementation](../../spikee/templates/basic_plugin.py) for details)

(NB, A dataset entry may contain a `exclude_from_transformations_regex` field, which contains strings that should not be transformed by a plugin. **This is a guideline and not respected by all plugins**)

## 4.3 Suggested Tasks
1. Generate a couple of datasets using different plugins and plugin piping. Look at how the resulting dataset entries are transformed.
2. (Optional) Create a `BasicPlugin` that reverses the input text, and generate a dataset using your plugin to see how it transforms the inputs.
3. Create a `Plugin` that implements [rot](https://en.wikipedia.org/wiki/ROT13), with a plugin option `shift` to specify the character shift. Ensure it adheres to the `exclude_from_transformations_regex` configuration. Generate a dataset using your plugin to see how it transforms the inputs.
